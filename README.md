## Цель работы
Целью работы является разработка веб-сервиса, позволяющего пользователям просматривать каталог гостиничных номеров, фильтровать номера по категориям, отправлять заявки на бронирование, а администратору — просматривать и обрабатывать поступившие заявки.

---

## Структура базы данных

### Таблица categories

Хранит категории номеров:

- category_id — идентификатор категории;
- name — название категории;
- color — цветовое оформление категории.

### Таблица rooms

Хранит данные о номерах:
- room_id — идентификатор номера;
- category_id — категория номера;
- title — название номера;
- price — цена за сутки;
image — изображение номера;
description — описание;
characteristics — характеристики номера.


### Таблица bookings

- booking_id — идентификатор заявки;
- room_id — выбранный номер;
- guest_name — имя гостя;
- guest_surname — фамилия гостя;
- phone — телефон;
- email — электронная почта;
- check_in — дата заезда;
- check_out — дата выезда;
- status — статус заявки;
- created_at — дата создания заявки.

## Серверная валидация данных

Были реализованы следующие проверки:
- имя и фамилия обязательны и могут содержать кириллицу, пробелы, точки и тире;
- телефон должен соответствовать формату +7(XXX)XXX-XX-XX;
- e-mail должен соответствовать формату электронной почты;
- дата заезда не может быть раньше текущей даты;
- дата выезда должна быть позже даты заезда;
- выбранный промежуток дат не должен пересекаться с уже существующими бронями на этот номер.

---

## Проверка занятости номера

Система проверяет, есть ли в таблице bookings уже существующие заявки на выбранный номер со статусом new или approved, даты которых пересекаются с новым бронированием.

Если пересечение найдено, заявка не создаётся, а пользователю выводится сообщение:
К сожалению, выбранный номер уже занят на указанные даты.

Для администратора была реализована панель управления сайтом.

## Безопасность
В проекте были реализованы базовые меры безопасности:

- использование prepared statements для SQL-запросов;
- экранирование выводимых данных через htmlspecialchars;
- серверная валидация всех данных формы;
- хеширование пароля администратора;
- проверка пароля через password_verify;
- использование сессий для защиты административной панели;
- обновление идентификатора сессии после успешного входа.
